# 투자 포트폴리오 트래커 — 프로젝트 플랜

## 프로젝트 개요

증권사 해외주식 거래내역 파일(.xls)을 파싱하여 종합거래내역 Google Sheet에 누적 저장하고, 이를 기반으로 모바일 웹 대시보드에서 포트폴리오 성과를 시각화하는 시스템.

## 핵심 워크플로우

```
[NH나무증권 .xls] → [파서 스크립트] → [종합거래내역 Google Sheet] → [웹 대시보드 시각화]
```

1. 증권사에서 해외주식거래내역 .xls 파일을 다운로드
2. 파서 스크립트가 .xls를 읽어서 종합거래내역 포맷으로 변환
3. 변환된 데이터를 Google Sheet(transactions 시트)에 append
4. 웹 대시보드에서 Google Sheet URL을 입력하면 자동으로 데이터를 fetch하여 시각화

## 입력 파일 규격

### NH나무증권 해외주식거래내역 (.xls)

- 파일명 포맷: `{증권사명}_{계좌번호}_{거래기간}`
  - 예시: `NH나무증권_202-01-292788_250101-250331`
- 실제 포맷: HTML 테이블 (EUC-KR 인코딩이라고 선언되지만 실제로는 UTF-8 + 깨진 한글)
- 구조: `<thead>` 2행(헤더) + `<tbody>` N행(데이터, 메인행 15컬럼 + 서브행 13컬럼이 교차)

#### 메인행 (15컬럼)
| 인덱스 | 컬럼명 | 설명 |
|--------|--------|------|
| 0 | 의결제일자 | YYYY.MM.DD |
| 1 | 거래유형 | 매도/매수/환전/입금/출금 등 (인코딩 깨짐) |
| 2 | 종목명 | IREN LIMITED ORD, ISHARES 0 TO 3 MNTH TREASURY BND ETF 등 |
| 3 | 수량 | 거래 수량 |
| 4 | 거래금액(외화) | USD 거래금액 |
| 5 | 외화잔고(전) | 거래 전 USD 잔고 |
| 6 | 정산금액(외화) | 정산된 금액 |
| 7 | 외화잔고(후) | 거래 후 USD 잔고 |
| 8 | 보유수량잔고 | 거래 후 해당 종목 보유수량 |
| 9 | 제보수(외화) | 수수료 |
| 10 | 거래수수료(외화) | 수수료 |
| 11 | 해외세금(외화) | 세금 |
| 12 | 거래세/소득세(외화) | 세금 |
| 13 | 통화 | USD 등 |
| 14 | 결제일자 | YYYY.MM.DD |

#### 서브행 (13컬럼, 메인행 바로 다음 행)
| 인덱스 | 컬럼명 | 설명 |
|--------|--------|------|
| 0 | 상세내역 | 원화환산매도/매수 등 (인코딩 깨짐) |
| 1 | 종목코드 | AU0000185993, US46436E7186 등 |
| 2 | 단가 | 주당 가격 |
| 3~11 | (원화 환산 데이터) | 메인행과 동일 구조 |
| 12 | 환율 | 거래 시점 USD/KRW 환율 |

#### 인코딩 깨짐 거래유형 매핑

파일이 EUC-KR 선언이지만 실제 바이트가 꼬여 있어서 아래 패턴으로 식별:

| 깨진 텍스트 패턴 | 실제 유형 |
|------------------|-----------|
| `ŵ` 포함 또는 `\xC5\xB5` | 매도 (SELL) |
| `ż` 포함 또는 `\xC5\xBC` | 매수 (BUY) |
| `ȯ` 포함 또는 `\xC8\xAF` | 환전 (FX) → 제외 |
| `Ա` 포함 또는 `\xD4\xB1` | 입금 (DEPOSIT) |
| `뱄` 또는 `킹` 포함 | 이체 (TRANSFER) → 제외 |

#### 종목명 → 티커 매핑

| 원본 종목명 | 매핑 티커 |
|-------------|-----------|
| IREN LIMITED ORD | IREN |
| ISHARES 0 TO 3 MNTH TREASURY BND ETF | SGOV |

새로운 종목이 추가되면 이 매핑 테이블을 확장해야 함.

## 종합거래내역 Google Sheet 포맷

- 시트 이름: `transactions`
- 컬럼 (A~K):

| 컬럼 | 이름 | 타입 | 설명 | 예시 |
|------|------|------|------|------|
| A | 거래일자 | YYYY-MM-DD | 거래일 | 2025-12-30 |
| B | 유형 | 텍스트 | 매수/매도/입금/출금/배당 | 매수 |
| C | 종목코드 | 텍스트 | 티커 심볼 (입금/출금은 빈값) | IREN |
| D | 수량 | 숫자 | 매수/매도 수량 | 30 |
| E | 단가USD | 숫자 | 주당 가격 (USD) | 40.00 |
| F | 금액USD | 숫자 | 총 거래금액 (USD) | 1200.00 |
| G | 환율 | 숫자 | 거래 시점 USD/KRW | 1433.9 |
| H | 금액KRW | 숫자 | 원화 금액 (=F*G 또는 원화 입금액) | 1720680 |
| I | 증권사 | 텍스트 | 파일명에서 추출 | NH나무증권 |
| J | 계좌번호 | 텍스트 | 파일명에서 추출 | 202-01-292788 |
| K | 비고 | 텍스트 | 메모 | 원화입금 |

- 제외 대상: 유형이 `환전` 또는 `이체`인 거래
- 날짜순 정렬 (오름차순)
- 중복 방지: (거래일자 + 유형 + 종목코드 + 수량 + 금액USD) 조합이 동일하면 skip

## 파서 스크립트 상세 (parse_namu.py)

### 입력
- resource/NH나무증권/{연도}/ 폴더 내 .xls 파일들

### 파일명 파싱
```
NH나무증권_202-01-292788_250101-250331.xls
→ 증권사: NH나무증권
→ 계좌번호: 202-01-292788
→ 거래기간: 2025-01-01 ~ 2025-03-31
```
파일명을 `_`로 split하여 첫 번째=증권사, 두 번째=계좌번호, 세 번째=기간

### 처리 로직
1. .xls 파일을 바이너리로 읽기 (실제로는 HTML)
2. BeautifulSoup으로 파싱 (from_encoding='euc-kr')
3. `<tbody>` 내 `<tr>` 순회
4. 메인행(15 td) + 서브행(13 td) 쌍으로 묶기
5. 거래유형 식별 (깨진 인코딩 패턴 매칭)
6. 환전/이체 제외
7. 종목명 → 티커 매핑
8. 종합거래내역 포맷으로 변환
9. output/종합거래내역.csv에 append (중복 제거)

### 실행 명령
```bash
# 특정 파일 파싱
python parser/parse_namu.py resource/NH나무증권/2026/NH나무증권_202-01-292788_260101-260331.xls

# 폴더 내 전체 파싱
python parser/parse_namu.py resource/NH나무증권/

# 전체 resource 스캔
python parser/parse_namu.py resource/
```

## 웹 대시보드 (web/index.html)

### 기술 스택
- HTML + CSS + Vanilla JavaScript (단일 파일, 프레임워크 없음)
- Chart.js 4.x (CDN)
- 모바일 퍼스트 반응형
- 다크 테마

### 데이터 소스
- Google Sheet URL 입력 → CSV export URL로 변환 → fetch → 파싱
- `https://docs.google.com/spreadsheets/d/{ID}/gviz/tq?tqx=out:csv&sheet=transactions`
- Google Sheet는 "링크가 있는 모든 사용자 뷰어" 권한 필요

### 프라이버시
- 서버 전송 없음 (모든 계산은 브라우저 내)
- localStorage 미사용 (탭 닫으면 데이터 소멸)
- 시세 업데이트 API에는 종목 티커(IREN 등)만 전송, 보유량/수익 미전송
- GitHub Pages에는 코드만 배포, 개인 데이터 없음

### 탭 구성

#### 1. 계좌현황 탭
- 요약 카드 4개: 총 평가액, 총 수익(수익률), 투자원금, 환율
- 월별 계좌 추세 차트 (평가액 vs 원금 라인차트)
- 월별 손익 바 차트
- 보유 종목 테이블 (종목, 수량, 현재가, 평가액, 비중%)

#### 2. 수익률 비교 탭
- 누적수익률 카드 3개: 내 포트폴리오 / S&P 500 / NASDAQ 100
- 월별 누적수익률 비교 라인차트 (3개 라인)
- 수익률 GAP 바 차트 (나 - S&P500, 나 - NASDAQ100)

#### 3. 거래내역 탭
- 전체 거래 테이블 (일자, 유형, 종목/계좌, 수량, 단가, 금액)
- 유형별 배지 컬러: 매수(보라), 매도(빨강), 입금/배당(초록)
- 증권사_계좌번호 표시

### 시세 업데이트
- "🔄 시세" 버튼 클릭 시 Anthropic API + 웹 검색으로 실시간 데이터 fetch
- 대상: 보유 종목 현재가, S&P500/NASDAQ100 월말 종가, USD/KRW 환율
- 벤치마크는 첫 거래월 전월 대비 누적수익률로 자동 계산

### 포트폴리오 계산 로직
1. 거래 데이터를 월별로 그룹핑
2. 각 월말 시점의 보유수량 계산 (매수 +qty, 매도 -qty)
3. 평균단가: (기존수량 × 기존평단 + 매수금액) / 신규수량
4. 월말 평가액: Σ(보유수량 × 월말가격 × 환율) + 현금잔고
5. 수익률: (평가액 / 투자원금 - 1) × 100
6. 벤치마크: 첫 거래 전월 종가 기준 누적 % 변화

## 구현 순서

### Phase 1: 파서 (parse_namu.py)
1. NH나무증권 .xls 파일 읽기 (HTML/EUC-KR 깨짐 처리)
2. 메인행+서브행 쌍 파싱
3. 거래유형 식별 (깨진 인코딩 패턴 매칭)
4. 종목명→티커 매핑
5. 환전/이체 제외
6. 파일명에서 증권사/계좌번호 추출
7. 종합거래내역 CSV 출력 (중복 제거, 날짜순 정렬)

### Phase 2: 웹 대시보드 (index.html)
1. Google Sheet URL 입력 → CSV fetch → 파싱
2. 포트폴리오 계산 엔진 (월별 보유현황, 평가액, 수익률)
3. 계좌현황 탭 (카드 + 추세 차트 + 손익 차트 + 보유종목 테이블)
4. 수익률 비교 탭 (벤치마크 라인차트 + GAP 차트)
5. 거래내역 탭 (거래 테이블)
6. 시세 업데이트 버튼 (Anthropic API + 웹검색)

### Phase 3: 확장 (향후)
- 추가 증권사 파서 (키움증권, 한국투자증권 등)
- 국내주식 거래내역 지원
- 종목별 상세 분석 (개별 종목 수익률, 매매 타이밍)
- 자산배분 파이차트
- 연간/분기별 수익률 리포트
